### 第一道门槛：寻找失踪的“COM18”

在 Windows 里，串口叫 `COM18`。但在 Ubuntu 里，所有的硬件设备都被挂载在 `/dev`（Device 设备）目录下。

**操作步骤：**

1. **先别插**你的 STM32（USB转TTL模块）。
    
2. 打开 Ubuntu 的终端（Terminal），输入这行探雷命令。
    
    ```
    ls /dev/ttyUSB* /dev/ttyACM*
    ```
    
    _（这时候可能会提示找不到文件，没关系，说明环境很干净。）_
    
3. **把 USB 模块插到电脑上**。
    
4. **再运行一次**上面那条命令！ 你会惊喜地发现，多出来了一个设备文件！通常它会长这样： 👉 **`/dev/ttyUSB0`** （或者 `/dev/ttyACM0`）
    

这就是你在 Ubuntu 里的 `COM18`！

> **黑客小技巧：** 如果你想看它是怎么被系统识别的，插上 USB 后立刻在终端输入 `dmesg | tail`，你会看到内核打印出的一堆硬件底层信息，非常硬核。

### 第二道门槛：打破 Linux 的“权限结界”（最容易踩坑！）

在 Windows 下，任何软件都能随便读写串口。但在 Linux 这种极其注重安全的系统里，**普通用户默认是没有权限读写硬件端口的！** 如果你现在直接去运行 Python 脚本，百分之百会报错：`Permission denied: '/dev/ttyUSB0'`。

我们有两种解法，我教你最正宗的“一劳永逸法”：

在 Ubuntu 终端里，输入这行命令，把你自己加入到拥有拨号和硬件读写权限的 `dialout` 用户组里：

```
sudo usermod -aG dialout $USER
```

_(输入你的 Ubuntu 开机密码确认。)_

🚨 **极其重要：** 加完用户组之后，权限不会立刻生效。你必须**注销当前 Ubuntu 用户并重新登录**（或者直接重启电脑），这个权限结界才会被彻底打破！
### 第三步：在 VS Code 里安营扎寨

现在，打开你 Ubuntu 里的 VS Code。

1. **准备 Python 环境：** 在 VS Code 顶部的菜单栏点击 `Terminal` -> `New Terminal`，确保你安装了 Python 的串口库。Ubuntu 里通常叫 `python3` 和 `pip3`：
    
    ```
    sudo apt update
    sudo apt install python3-pip
    pip3 install pyserial
    ```
    
2. **微调代码，接管硬件：** 把你刚才在 Windows 下写的完整 Python 代码复制到 VS Code 里新建的 `.py` 文件中。 **只需要改动一行代码！** 将全局配置里的端口号换成你刚才查到的设备名：

    ```
    # ==========================================
    # 1. 打开串口联络通道 (全局配置)
    # ==========================================
    PORT = '/dev/ttyUSB0'  # <--- 把 'COM18' 换成这个！
    BAUDRATE = 115200
    ```

### 见证跨平台奇迹

在 VS Code 的终端里，运行你的脚本：
```
python3 你的文件名.py
```

如果你看到那句熟悉的： `串口连接成功！机器已接管。` 并且屏幕下方立刻跳出了那个狂飙的： `[底层雷达] 状态: 停机待命 | 实时 ARR 速度值: 1200`
