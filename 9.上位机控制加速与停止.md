```python
import serial

import time

import threading # 引入多线程魔法库

  

# ==========================================

# 1. 打开串口联络通道 (全局配置)

# ==========================================

PORT = 'COM18'     # 请务必改成你真实的串口号！

BAUDRATE = 115200

  

print(f"尝试连接串口 {PORT} ...")

try:

    ser = serial.Serial(PORT, BAUDRATE, timeout=1)

    print(" 串口连接成功！机器已接管。")

except Exception as e:

    print(f" 串口打开失败: {e}")

    exit()

  

# ==========================================

# 2. 核心功能函数：打包并发送任意指令

# ==========================================

def send_command(cmd, arr_val):

    """

    专门负责把指令和速度打包成 7 字节装甲包裹，并发送出去。

    """

    arr_h = (arr_val >> 8) & 0xFF

    arr_l = arr_val & 0xFF

    checksum = (cmd + arr_h + arr_l) & 0xFF

    packet = [0xAA, 0x55, cmd, arr_h, arr_l, checksum, 0xFF]

    byte_data = bytes(packet)

    ser.write(byte_data)

    # 为了保持屏幕清爽，我们只打印业务包，不打印心跳包

    if cmd != 0x00:

        hex_string = ' '.join([f'{b:02X}' for b in byte_data])

        print(f"   [发送底层数据] -> {hex_string}")

  

# ==========================================

# 3. 后台锅炉工（独立的心跳线程）

# ==========================================

def heartbeat_task():

    """

    这个函数会在一个独立的隐形时空里运行，永远不死循环。

    它唯一的任务：每隔 1 秒，默默发送一次 0x00 心跳包。

    """

    while True:

        # 发送心跳包: 指令 0x00, 参数 0

        send_command(0x00, 0)

        time.sleep(1) # 锅炉工睡 1 秒，醒来接着发

  

# 启动后台锅炉工！(daemon=True 表示主程序退出时，锅炉工也跟着下班)

heartbeat_thread = threading.Thread(target=heartbeat_task, daemon=True)

heartbeat_thread.start()

print("后台心跳保命机制已启动...")

  

# ==========================================

# 4. 前台接待员（主线程交互逻辑）

# ==========================================

print("\n" + "="*40)

print("  传送带超级控制台 ")

print("输入 'A 800'  -> 启动并加速到 ARR=800")

print("输入 'S'      -> 紧急平稳刹车")

print("输入 'Q'      -> 退出程序")

print("="*40 + "\n")

  

try:

    # 前台永远在这里等你发号施令

    while True:

        user_input = input("\n请输入指令 >>> ").strip().upper()

        if user_input == 'Q':

            print("正在退出...")

            break # 打破死循环，结束程序

        elif user_input == 'S':

            print("执行紧急刹车！")

            send_command(0x03, 0) # 0x03 是刹车指令，参数填 0 就行

        elif user_input.startswith('A '):

            # 尝试从输入的字符串里提取出速度数字

            try:

                # 把 'A 800' 按空格切开，取第二部分 '800'，并转成整数

                speed_str = user_input.split(' ')[1]

                target_speed = int(speed_str)

                # 安全限制：防止你输入太离谱的数字把定时器搞崩溃

                if target_speed < 99 or target_speed > 5000:

                    print("速度参数越界！请输入 99 到 5000 之间的数字。")

                    continue

                print(f" 执行启动/变压，目标速度设定为 ARR={target_speed}")

                send_command(0x01, target_speed) # 发送 0x01 指令和速度

            except ValueError:

                print("指令格式错误！正确格式例如: A 800")

        else:

            print("未知指令。")

  

except KeyboardInterrupt:

    # 捕捉你按 Ctrl+C 强行退出的动作，优雅收场

    print("\n程序被强制中断。")

  

# ==========================================

# 5. 打烊收工

# ==========================================

ser.close()

print("串口已安全关闭。拜拜！")
```
- **绝对的安静与稳定：** 控制台不会被疯狂刷屏的心跳包淹没。但是你去看单片机，它的看门狗永远不会咬人，电机永远不会因为超时而莫名其妙地断电重启。这是因为**后台子线程**（`heartbeat_thread`）正在默默地为你兜底。
    
- **随心所欲的指挥感：** * 你输入 `A 1200`，回车。电机极其缓慢地动起来。
    
    - 你不用等它停，立刻输入 `A 500`，回车。你会听到电机发出极其丝滑的加速声，声音变尖锐！
        
    - 你再次输入 `A 99`，回车。电机瞬间进入最高速狂飙！
        
    - 你敲下 `S`，回车。电机乖乖地进入完美减速斜坡，稳稳停下。


- **大白话：** Python 默认是单线程的（就像店里只有一个老板，如果去接待客人，就没法去锅炉房添煤）。引入 `threading`，就等于学会了“分身术”，可以同时干两件完全互不干扰的事情。
    
### 2. 打造“后台锅炉工”（独立时空）

```
def heartbeat_task():
    while True:
        send_command(0x00, 0)
        time.sleep(1) # 锅炉工睡 1 秒，醒来接着发
```

- **大白话：** 我们给分身（锅炉工）写了一份专门的岗位说明书。
    
- `while True:`：这是一个死循环。但别怕，因为它是跑在分身里的，所以**绝对不会**卡住你的主程序。
    
- `time.sleep(1)`：极其重要！发完一次心跳包后，强制让这个分身“睡” 1 秒钟。如果不加这句，分身会以每秒几百万次的速度疯狂发心跳包，电脑 CPU 瞬间会卡爆，单片机也会被撑死。

我们回头看看是怎么定义这个发快递的函数的：

```
def send_command(cmd, arr_val):
```

这个定义说明，这个函数**必须、强制**要求你每次调用它时，提供**两个**参数：一个是指令（`cmd`），一个是速度值（`arr_val`）。 如果你只写 `send_command(0x00)`，Python 就会立刻报错罢工：“老板，你让我发快递，给了指令，但没给我速度参数啊！” 所以，为了不让 Python 报错，即使我们不需要速度，也必须随便填一个数字给它“占个坑”。填 `0` 是最符合人类直觉的。

### 第二层：工业协议的“死规矩”（凑字数）

还记得咱们定下的“7字节装甲包裹”协议吗？ `[帧头AA, 帧头55, 指令, 高八位, 低八位, 校验和, 帧尾FF]`

这就像顺丰快递的**标准尺寸一号箱**。不管你里面装的是大石头（调速指令）还是一根羽毛（心跳包），箱子的尺寸是**固定死**的，快递单上的格子也必须填满。

当 `arr_val = 0` 时，Python 内部是这样切分的：

- `0` 的高八位是 `0x00`
    
- `0` 的低八位也是 `0x00`
    

所以，最终发出去的心跳包裹长这样： `AA 55 00 00 00 00 FF` 你看，这个包裹完完整整地凑齐了 7 个字节。如果你不传那个 `0`，包裹就少了两个字节，下位机（STM32）的安检机一看尺寸不对，直接就当成垃圾扔了！

```
heartbeat_thread = threading.Thread(target=heartbeat_task, daemon=True)
```

- **大白话：** 真正用泥巴捏出一个分身人偶（Thread对象）。
    
- `target=heartbeat_task`：把上面写好的岗位说明书塞进他的脑子里。
    
- **`daemon=True`（全场最关键的参数）：** `daemon` 叫“守护线程”。它的意思是：**你是附属品，如果老板（主程序）下班关门了，你必须立刻跟着死掉，不能一个人留在店里瞎捣鼓。** 如果不加这个，你关了程序，这个隐形人还在后台偷偷运行，非常可怕。

```
heartbeat_thread.start()
```

- **大白话：** 在分身人偶上踢一脚，大喊一声“干活去！”。从这一行代码开始，这个分身就进入了一个平行的独立时空，开始每秒默默发心跳包了。程序立刻往下走，不管他了。

### 3. “前台接待员”（主程序的死循环）

锅炉工去后台了，现在轮到我们主程序在前面接待你了。

```
    # 前台永远在这里等你发号施令
    while True:
        user_input = input("\n请输入指令 >>> ").strip().upper()
```

- **`input(...)`：** 这是一个“阻塞”函数。程序走到这里会**死死停住**，屏幕上光标闪烁，等你敲键盘回车。_(注意：此时主程序停住了，但后台的锅炉工因为在平行时空，依然在每秒发心跳！这就是多线程的奇迹！)_
    
- **`.strip().upper()`：** 贴心的小优化。`.strip()` 会把你手抖多打的空格切掉；`.upper()` 会把你输入的小写字母 `a 800` 强行变成大写的 `A 800`，省得后面判断大小写。
    

```
        elif user_input.startswith('A '):
```

- **大白话：** 判断你输入的内容是不是以 `A` （A加一个空格）开头的。
    

```
            try:
                speed_str = user_input.split(' ')[1] 
                target_speed = int(speed_str)
```

- **极其巧妙的字符串切割 (`split`)：**
    
    - 假设你输入了 `"A 1500"`。
        
    - `.split(' ')` 就像一把刀，沿着空格把这句话切成两段，变成一个列表：`["A", "1500"]`。
        
    - `[1]` 取出列表的第 2 个元素（编程从 0 开始数），也就是取出了 `"1500"` 这个纯文本。
        
    - `int(...)`：把文本的 `"1500"` 真正转换成数学上可以用来计算的整数 `1500`。
        
- **内层 `try...except ValueError`：** 防呆设计。如果你捣乱，输入了 `A 哈哈哈`，`int("哈哈哈")` 就会报错。加了 `try`，程序就不会崩溃，而是礼貌地提示你“指令格式错误”。
    

```
                # 安全限制
                if target_speed < 99 or target_speed > 5000:
                    print("速度参数越界！...")
                    continue
```

- **`continue`：** 这个词的意思是“别往下走了，立刻倒带回 `while True` 的开头，重新问客户要指令”。这是保护下位机定时器不被超出范围的极值搞崩溃的重要防线。
    

```
except KeyboardInterrupt:
    # 捕捉你按 Ctrl+C 强行退出的动作，优雅收场
    print("\n程序被强制中断。")
```

- **大白话：** 在命令行里跑 Python 时，按下 `Ctrl + C` 是强制杀掉程序的快捷键。如果不加这个拦截，屏幕上会爆出一大堆红色的丑陋错误代码。有了这行，它就能优雅地鞠个躬再说再见。
