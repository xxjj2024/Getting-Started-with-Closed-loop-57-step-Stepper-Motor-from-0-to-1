### 第一步：ROS 2 架构大转换（思想升级）

在原来的 Python 脚本里，你必须自己死死盯着键盘（`input()`）。 但在 ROS 2 的世界里，我们的代码将升级为一个**“节点 (Node)”**。

在 ROS 2 里，这台电机的架构是这样的：

1. **订阅者 (Subscriber)：** 代替原来的 `input()`。它竖起耳朵，专门监听 ROS 2 网络里叫 `motor_cmd`（电机指令）的频道。别的程序往这个频道发字符串 `"A 800"`，它就立刻转发给 STM32。
    
2. **发布者 (Publisher)：** 代替原来的 `print()` 仪表盘。单片机发回来的速度报告，它不再只是打印在屏幕上，而是打包成消息，大声广播到 `motor_status` 频道。这样，你的其他节点（比如未来的 AI 视觉抓取节点）就能直接读取到电机的当前转速！
    
3. **定时器 (Timer)：** 代替原来的锅炉工 `time.sleep(1)`。ROS 2 自带极其精准的定时器，让它每隔 1 秒自动调一次心跳函数。

### 第二步：编写你的第一个 ROS 2 硬件驱动节点

进入你的 ROS 2 工作空间（比如你截图里的 `ros2_21_tutorials/motorcontrol`），新建一个文件叫 `motor_driver_node.py`，把下面这段**融合了串口通信和 ROS 2 机制**的代码填进去：

```Python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String # 我们先用最简单的字符串消息来通讯
import serial
import threading
import time

class MotorDriverNode(Node):
    def __init__(self):
        super().__init__('motor_driver_node') # 给这个节点起个响亮的名字
        
        # 1. 初始化串口 (记得改成你虚拟机里真实查到的端口！)
        self.port = '/dev/ttyUSB0' 
        self.baudrate = 115200
        try:
            self.ser = serial.Serial(self.port, self.baudrate, timeout=0.1)
            self.get_logger().info(f"✅ 成功接管电机底层串口: {self.port}")
        except Exception as e:
            self.get_logger().error(f"❌ 串口打开失败: {e}")
            exit()

        # 2. 成立通信部门 (ROS 2 发布者与订阅者)
        # 发布者：专门把单片机发来的状态，广播到 '/motor_status' 频道
        self.status_pub = self.create_publisher(String, 'motor_status', 10)
        
        # 订阅者：专门监听外界发到 '/motor_cmd' 频道的指令
        self.cmd_sub = self.create_subscription(String, 'motor_cmd', self.cmd_callback, 10)
        
        # 3. 雇佣定时锅炉工 (ROS 2 定时器，每 1.0 秒触发一次心跳)
        self.heartbeat_timer = self.create_timer(1.0, self.heartbeat_callback)
        
        # 4. 派分身去死磕串口接收 (依然使用独立线程，不阻塞 ROS 2 主循环)
        self.receive_thread = threading.Thread(target=self.receive_task, daemon=True)
        self.receive_thread.start()

    # ==========================================
    # 核心动作：打包发送底层十六进制
    # ==========================================
    def send_command(self, cmd, arr_val):
        arr_h = (arr_val >> 8) & 0xFF
        arr_l = arr_val & 0xFF
        checksum = (cmd + arr_h + arr_l) & 0xFF 
        packet = [0xAA, 0x55, cmd, arr_h, arr_l, checksum, 0xFF]
        self.ser.write(bytes(packet))

    # 定时器回调：发心跳
    def heartbeat_callback(self):
        self.send_command(0x00, 0)

    # ==========================================
    # 订阅者回调：有人在频道里下达了指令！
    # ==========================================
    def cmd_callback(self, msg):
        command_str = msg.data.strip().upper()
        self.get_logger().info(f"收到上层 ROS 指令: {command_str}")
        
        if command_str == 'S':
            self.send_command(0x03, 0) 
        elif command_str.startswith('A '):
            try:
                target_speed = int(command_str.split(' ')[1])
                if 99 <= target_speed <= 5000:
                    self.send_command(0x01, target_speed) 
            except ValueError:
                self.get_logger().warn("指令格式错误，丢弃！")

    # ==========================================
    # 接收线程：滑动窗口解析，并转换成 ROS 消息广播！
    # ==========================================
    def receive_task(self):
        buffer = bytearray()
        while rclpy.ok(): # 只要 ROS 没关机，就一直循环
            if self.ser.in_waiting > 0:
                buffer.extend(self.ser.read(self.ser.in_waiting)) 
                
                while len(buffer) >= 8:
                    if buffer[0] == 0xAA and buffer[1] == 0x55 and buffer[2] == 0x81:
                        calc_sum = (buffer[2] + buffer[3] + buffer[4] + buffer[5]) & 0xFF
                        if calc_sum == buffer[6] and buffer[7] == 0xFF:
                            
                            motor_state = buffer[3]
                            current_speed = (buffer[4] << 8) | buffer[5]
                            
                            state_str = "停机"
                            if motor_state == 1: state_str = "加速"
                            elif motor_state == 2: state_str = "匀速"
                            elif motor_state == 3: state_str = "减速"
                            
                            # 【核心魔法】：打包成 ROS 2 的 String 消息，广播出去！
                            msg = String()
                            msg.data = f"状态:{state_str}, 实时ARR:{current_speed}"
                            self.status_pub.publish(msg)
                            
                            buffer = buffer[8:]
                        else:
                            buffer.pop(0)
                    else:
                        buffer.pop(0)
            time.sleep(0.01)

def main(args=None):
    rclpy.init(args=args)
    node = MotorDriverNode()
    
    try:
        # 这一句相当于启动了 ROS 2 的“总指挥”，它会接管所有定时器和订阅事件
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.ser.close()
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 第三步：见证 ROS 2 的分布式魔法

如果你的 USB 端口搞定了，直接在终端里用 `python3 motor_driver_node.py` 运行它。

此时，代码看起来仿佛“卡住”了，什么都没输出。 **别慌！这就是 ROS 2 的精髓——它在后台静静地工作。**

现在，我们再开 **两个新的 Ubuntu 终端**（相当于雇了两个人来和你的电机互动）：

**在终端 2（监控者）：** 输入命令去监听状态频道：
```
ros2 topic echo /motor_status
```

你会震撼地看到，屏幕开始疯狂刷出单片机传回来的真实状态消息！ `data: '状态:停机, 实时ARR:1200'`

**在终端 3（指挥官）：** 输入命令，通过 ROS 2 网络下发加速指令：

```
ros2 topic pub --once /motor_cmd std_msgs/msg/String "{data: 'A 800'}"
```

当你按下回车的那一瞬间：

1. **终端 3** 把 `'A 800'` 这个字符串扔到了空中。
    
2. 你的 **Python 驱动节点** 瞬间抓住了它，翻译成 `AA 55 01 03 20 24 FF`，顺着串口线砸给 STM32。
    
3. **STM32** 执行加速，并把速度实时回传。
    
4. **终端 2** 的屏幕上，你会亲眼看着传回来的速度从 1200 极速下降到 800，状态变成“匀速”！
