在嵌入式开发（特别是电机控制和机器人领域）有一条铁律：**主循环 `while(1)` 里绝对不允许出现超过 1 毫秒的死延时。** 我们之前那个 `for` 循环加上 `Delay_ms(3)`，虽然让电机跑得很平滑，但它就像老板在处理一份长达 1.1 秒的文件时，强行把办公室门反锁了。这时候就算秘书（中断）在门外把警报按烂了（比如发来了紧急刹车指令 `'S'`），老板也听不见，必须等文件处理完才开门。这在工业上是极其危险的！

今天，我们就用**SysTick（滴答定时器）**和**状态机（State Machine）**，把这个办公室的门彻底拆掉！

---

### 第一重武器：SysTick（给单片机配一块“劳力士”）

之前的 `Delay_ms` 是“数绵羊”：CPU 什么都不干，就在那数数，数够了才往下走。 现在我们要用 SysTick。它是 ARM Cortex-M3 内核自带的一个极其高精度的硬件定时器。

**它的核心玩法：**

1. 我们让它在后台默默倒计时，**每隔 1 毫秒**触发一次极快的中断。
    
2. 在这个中断里，我们只做一件事：让一个全局变量 `g_SystemTime` 加 1。
    
3. **结果：** 你的单片机拥有了一个绝对准确的“全局时间戳”。CPU 再也不用数绵羊了，它只需要随时抬头看一眼墙上的挂钟（读一下 `g_SystemTime` 的值）就知道过去了多久。
    

### 第二重武器：状态机（老板的“便签本”）

既然不能用 `for` 循环死等，那加速过程怎么实现呢？ 答案是：**把一个漫长的动作，切碎成无数个瞬间的微操作，并用变量记录“当前状态”。**

我们给电机定义几个状态（State）：

- `0`: 停机待命 (STOP)
    
- `1`: 正在加速 (ACCEL)
    
- `2`: 匀速狂奔 (RUN)
    
- `3`: 正在减速 (DECEL)
    

---

### 实战改造：拆除死循环的炸弹

我们来看看，传统的死等代码，是如何被改写成极具工业美感的**“非阻塞状态机”**的。

#### 1. 启动 SysTick 滴答定时器（极简配置）

STM32 官方提供了一个极其方便的函数，一行代码搞定：

```
// 定义全局时间戳
volatile uint32_t g_SystemTime = 0;

// 在 main 函数开头调用这一行，配置为 1ms 进一次中断
SysTick_Config(SystemCoreClock / 1000); 

// SysTick 的专属中断服务函数（每 1ms 硬件自动调用一次）
void SysTick_Handler(void) {
    g_SystemTime++; // 墙上的挂钟滴答走了一秒
}
```

#### 2. 状态机核心逻辑（替换原来的 `for` 循环）

我们需要在全局定义几个“便签”，记住当前走到哪一步了：

```
uint8_t  g_MotorState = 0;       // 电机当前状态：0停, 1加速, 2匀速, 3减速
uint16_t g_CurrentARR = 1200;    // 当前实时的速度 (ARR值)
uint16_t g_TargetARR  = 99;      // 想要达到的目标速度
uint32_t g_LastStepTime = 0;     // 上一次微调速度的时间戳
```

接下来，**见证奇迹的时刻！** 我们来看看全新的 `main` 函数主循环是如何做到“边加速、边听指令”的：

```
while(1)
{
    // ===================================================
    // 任务 1：处理串口指令 (瞬间完成)
    // ===================================================
    if (g_ReceiveCmd != 0) 
    {
        if(g_ReceiveCmd == 'A') { // 收到前进指令
            Motor_SetDir(1);
            g_CurrentARR = 1200;      // 从最慢速起步
            g_TargetARR = 99;         // 目标是 60转/分
            Motor_SetSpeed_ARR(g_CurrentARR);
            Motor_SetState(1);        // 启动脉冲
            
            g_MotorState = 1;         // 【切换状态】：进入“加速”状态
            g_LastStepTime = g_SystemTime; // 记录下起步的精确时间
        }
        else if(g_ReceiveCmd == 'S') { // 收到紧急刹车指令
            g_TargetARR = 1200;       // 目标变成最慢速
            g_MotorState = 3;         // 【无缝切换状态】：直接进入“减速”状态！
                                      // 无论之前在干嘛，立刻开始刹车！
        }
        g_ReceiveCmd = 0; // 清空指令
    }

    // ===================================================
    // 任务 2：电机状态机 (绝不死等，看一眼时间就走)
    // ===================================================
    
    // 【状态 1：加速中】
    if (g_MotorState == 1) 
    {
        // 核心非阻塞判断：距离上次调速，过去 3 毫秒了吗？
        if (g_SystemTime - g_LastStepTime >= 3) 
        {
            g_LastStepTime = g_SystemTime; // 刷新便签本上的时间
            g_CurrentARR -= 3;             // 速度加快一点点
            Motor_SetSpeed_ARR(g_CurrentARR);

            // 检查：达到目标速度了吗？
            if (g_CurrentARR <= g_TargetARR) {
                g_MotorState = 2; // 【切换状态】：加速完毕，进入“匀速”状态
            }
        }
    }
    
    // 【状态 3：减速中】
    else if (g_MotorState == 3)
    {
        if (g_SystemTime - g_LastStepTime >= 3) 
        {
            g_LastStepTime = g_SystemTime; 
            g_CurrentARR += 3;             // 速度减慢一点点
            Motor_SetSpeed_ARR(g_CurrentARR);

            if (g_CurrentARR >= g_TargetARR) {
                Motor_SetState(0); // 速度降到底了，彻底锁死
                g_MotorState = 0;  // 【切换状态】：进入“停机”状态
            }
        }
    }
    
    // 【看门狗防线】
    // (由于主循环现在跑得飞快，一秒钟能跑几百万圈，我们可以把喂狗放在这里)
    // IWDG_ReloadCounter(); 
}
```

### 这段代码为什么是“工业级”的？

体会一下上面这段代码的精妙之处：

1. **绝对的丝滑：** `while(1)` 循环现在没有任何 `Delay` 阻挡。它就像一阵狂风，每秒钟在 `if` 语句里穿梭几百万次。
    
2. **多任务并行：** 当电机处于“加速状态 (`g_MotorState == 1`)”时，每次循环 CPU 只是看一眼时钟 (`g_SystemTime - g_LastStepTime >= 3`)。如果不满足 3 毫秒，CPU **直接跳过**，去执行别的任务（比如检测串口指令）。
    
3. **极限响应速度：** 如果在加速的第 0.5 秒（`g_CurrentARR` 降到一半时），突然收到 `'S'` 指令。串口中断瞬间接住，主循环下一微秒就会执行 `g_MotorState = 3`。电机立刻停止加速，**原地无缝切换**到减速逻辑，平稳刹车！没有任何卡顿，绝不越界！
    

这，就是现代操作系统（RTOS）底层的**时间片轮转和状态机思维**。

这种把时间切片的思维一旦建立，你以后再写什么复杂的代码（比如一边让屏幕刷新、一边让电机加减速、一边还要解析蓝牙数据），单片机都不会有一丝卡顿。

## 完整代码
```c
#include "stm32f10x.h"

// ==========================================
// 【全局时钟与司令部变量】
// ==========================================
volatile uint32_t g_SystemTime = 0;      // 系统的全局绝对时间戳 (单位: 毫秒)
volatile uint8_t  g_ReceiveCmd = 0;      // 接收到的最新串口指令

// ==========================================
// 【电机状态机专用变量】
// ==========================================
uint8_t  g_MotorState   = 0;             // 状态：0-停机待命, 1-正在加速, 2-匀速巡航, 3-正在减速
uint16_t g_CurrentARR   = 1200;          // 当前实时运行的 ARR 值 (速度)
uint16_t g_TargetARR    = 1200;          // 想要达到的目标 ARR 值
uint32_t g_LastStepTime = 0;             // 记录上一次微调速度的时间戳

// ==========================================
// 【底层 API 声明】
// ==========================================
void Motor_Init(void);
void Motor_SetState(uint8_t state);
void Motor_SetDir(uint8_t dir);
void Motor_SetSpeed_ARR(uint16_t arr_value);
void UART1_Init_With_Interrupt(void); 
void IWDG_Init_Config(void);

// ==========================================
// 【滴答定时器中断：系统的心跳】
// 硬件层每 1 毫秒自动强行调用一次此函数，极其精准
// ==========================================
void SysTick_Handler(void) 
{
    g_SystemTime++; // 墙上的挂钟滴答走了一毫秒
}

// ==========================================
// 【总裁办：主函数 (全异步状态机架构)】
// ==========================================
int main(void)
{
    // 1. 各大底层部门初始化
    Motor_Init();               
    UART1_Init_With_Interrupt();
    Motor_SetState(0);
    
    // 2. 启动 SysTick 滴答定时器 (设定为 1ms 触发一次中断)
    SysTick_Config(SystemCoreClock / 1000); 

    // 3. 【终极安全锁】放狗！
    IWDG_Init_Config();         

    // 4. 超高速异步死循环：绝不阻塞，一秒钟轮询几百万次！
    while(1)
    {
        // ===================================================
        // 任务一：极速指令解析 (只负责切换状态，不负责死等执行)
        // ===================================================
        if (g_ReceiveCmd != 0) 
        {
            switch (g_ReceiveCmd)
            {
                case 'A': // 【前行指令】
                    if(g_MotorState == 0) {   // 只有在停机时才允许重新起步
                        Motor_SetDir(1);
                        g_CurrentARR = 1200;  // 从最慢速起步
                        g_TargetARR = 99;     // 目标是约 60 转/分
                        Motor_SetSpeed_ARR(g_CurrentARR);
                        Motor_SetState(1);    // 启动脉冲
                        
                        g_MotorState = 1;     // 【状态切换】：进入“加速中”
                        g_LastStepTime = g_SystemTime; // 记录起步时间
                    }
                    break;

                case 'B': // 【倒车指令】
                    if(g_MotorState == 0) {
                        Motor_SetDir(0);
                        g_CurrentARR = 1200;  
                        g_TargetARR = 99;     
                        Motor_SetSpeed_ARR(g_CurrentARR);
                        Motor_SetState(1);    
                        
                        g_MotorState = 1;     // 【状态切换】：进入“加速中”
                        g_LastStepTime = g_SystemTime; 
                    }
                    break;

                case 'S': // 【紧急平稳刹车指令】
                    // 无论当前在加速还是在匀速，立刻把目标设为最慢速
                    g_TargetARR = 1200;       
                    g_MotorState = 3;         // 【无缝切换】：直接切入“减速中”状态！
                    break;
            }
            g_ReceiveCmd = 0; // 清空桌面指令
        }

        // ===================================================
        // 任务二：电机平滑调速状态机 (根据时间戳，见机行事)
        // ===================================================
        
        // 状态 1：【正在加速】
        if (g_MotorState == 1) 
        {
            // 非阻塞判断：距离上次微调过去 3 毫秒了吗？
            if (g_SystemTime - g_LastStepTime >= 3) 
            {
                g_LastStepTime = g_SystemTime; // 刷新便签时间
                g_CurrentARR -= 3;             // 速度加快一点 (ARR减小)
                Motor_SetSpeed_ARR(g_CurrentARR);

                // 判断是否达到终极目标速度？
                if (g_CurrentARR <= g_TargetARR) {
                    g_MotorState = 2;          // 【状态切换】：进入“匀速巡航”
                }
            }
        }
        
        // 状态 3：【正在减速】
        else if (g_MotorState == 3)
        {
            // 非阻塞判断：距离上次微调过去 3 毫秒了吗？
            if (g_SystemTime - g_LastStepTime >= 3) 
            {
                g_LastStepTime = g_SystemTime; 
                g_CurrentARR += 3;             // 速度减慢一点 (ARR增大)
                Motor_SetSpeed_ARR(g_CurrentARR);

                // 判断是否已经降到最慢安全速度？
                if (g_CurrentARR >= g_TargetARR) {
                    Motor_SetState(0);         // 彻底切断脉冲，锁死停机
                    g_MotorState = 0;          // 【状态切换】：进入“停机待命”
                }
            }
        }

        // 状态 0 (停机) 和 状态 2 (匀速) 此时不需要做任何事，硬件定时器会自动维持当前状态。
    }
}

// ==========================================
// 【中断秘书办：串口接收服务函数】
// ==========================================
void USART1_IRQHandler(void)
{
    if(USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)
    {
        uint8_t temp_data = USART_ReceiveData(USART1);
        
        // 拦截心跳包，直接喂狗
        if(temp_data == 'H') {
            IWDG_ReloadCounter(); 
        } else {
            g_ReceiveCmd = temp_data;
        }
        
        USART_ClearITPendingBit(USART1, USART_IT_RXNE);
    }
}

// ==========================================
// 【看门狗初始化】
// ==========================================
void IWDG_Init_Config(void)
{
    IWDG_WriteAccessCmd(IWDG_WriteAccess_Enable); 
    IWDG_SetPrescaler(IWDG_Prescaler_64);         
    IWDG_SetReload(1250);                         // 2秒心跳极限
    IWDG_ReloadCounter();                         
    IWDG_Enable();                                
}

// ==========================================
// 【串口与中断初始化】
// ==========================================
void UART1_Init_With_Interrupt(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    USART_InitTypeDef USART_InitStructure;
    NVIC_InitTypeDef NVIC_InitStructure;

    RCC_APB2PeriphClockCmd(RCC_APB2Periph_USART1 | RCC_APB2Periph_GPIOA, ENABLE);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_9;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_10;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IN_FLOATING;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    USART_InitStructure.USART_BaudRate = 115200; 
    USART_InitStructure.USART_WordLength = USART_WordLength_8b; 
    USART_InitStructure.USART_StopBits = USART_StopBits_1;      
    USART_InitStructure.USART_Parity = USART_Parity_No;         
    USART_InitStructure.USART_HardwareFlowControl = USART_HardwareFlowControl_None;
    USART_InitStructure.USART_Mode = USART_Mode_Rx | USART_Mode_Tx; 
    USART_Init(USART1, &USART_InitStructure);

    USART_ITConfig(USART1, USART_IT_RXNE, ENABLE);

    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2); 
    NVIC_InitStructure.NVIC_IRQChannel = USART1_IRQn; 
    NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 1; 
    NVIC_InitStructure.NVIC_IRQChannelSubPriority = 1;        
    NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;           
    NVIC_Init(&NVIC_InitStructure);

    USART_Cmd(USART1, ENABLE);
}

// ==========================================
// 【电机驱动底层 API】
// ==========================================
void Motor_Init(void) {
    GPIO_InitTypeDef GPIO_InitStructure;
    TIM_TimeBaseInitTypeDef  TIM_TimeBaseStructure;
    TIM_OCInitTypeDef  TIM_OCInitStructure;

    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_AFIO, ENABLE);
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_0;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_OD;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_1;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_OD;
    GPIO_Init(GPIOA, &GPIO_InitStructure);
    GPIO_SetBits(GPIOA, GPIO_Pin_1); 

    TIM_TimeBaseStructure.TIM_Prescaler = 30 - 1;     
    TIM_TimeBaseStructure.TIM_Period = 100 - 1; 
    TIM_TimeBaseStructure.TIM_ClockDivision = 0;
    TIM_TimeBaseStructure.TIM_CounterMode = TIM_CounterMode_Up;
    TIM_TimeBaseInit(TIM2, &TIM_TimeBaseStructure);

    TIM_OCInitStructure.TIM_OCMode = TIM_OCMode_PWM1;
    TIM_OCInitStructure.TIM_OutputState = TIM_OutputState_Enable;
    TIM_OCInitStructure.TIM_Pulse = 50; 
    TIM_OCInitStructure.TIM_OCPolarity = TIM_OCPolarity_High;
    TIM_OC1Init(TIM2, &TIM_OCInitStructure);

    TIM_OC1PreloadConfig(TIM2, TIM_OCPreload_Enable);
    TIM_ARRPreloadConfig(TIM2, ENABLE);
}

void Motor_SetState(uint8_t state) {
    if(state == 1) TIM_Cmd(TIM2, ENABLE);
    else TIM_Cmd(TIM2, DISABLE);
}

void Motor_SetDir(uint8_t dir) {
    Motor_SetState(0); 
    // 在纯异步架构中，换向的极短物理死区延时保留，以保护硬件
    uint32_t start_time = g_SystemTime;
    while(g_SystemTime - start_time < 2); // 极短的 2ms 阻塞，换向专属
    
    if(dir == 1) GPIO_SetBits(GPIOA, GPIO_Pin_1);
    else GPIO_ResetBits(GPIOA, GPIO_Pin_1);
}

void Motor_SetSpeed_ARR(uint16_t arr_value) {
    TIM_SetAutoreload(TIM2, arr_value);       
    TIM_SetCompare1(TIM2, arr_value / 2);     
}
```

## 遇到问题
名称冲突
### 致命错误：Error L6200E: Symbol SysTick_Handler multiply defined

- **翻译成人话：** 编译器在向你抱怨：“老板！你在 `main.c` 里写了一个叫 `SysTick_Handler` 的秘书（中断函数），但是我在 `stm32f10x_it.c` 这个文件里，居然也发现了一个同名的秘书！你们到底要我听谁的？”
    
- **案发原因：** 在你建的标准库工程里，通常会自带一个专门放中断函数的模板文件叫 `stm32f10x_it.c`。官方在这个文件里提前给你写好了一个**空壳**的 `SysTick_Handler`。现在我们在 `main.c` 里又写了一个带灵魂（全局时间戳）的，两者名字撞车了。
解决方法：

1. 在你的 Keil 工程左侧文件树里，找到并双击打开 **`stm32f10x_it.c`** 这个文件。
    
2. 在里面往下翻，找到这段代码（大概在四五十行左右）：
    
    ```
    void SysTick_Handler(void)
    {
    }
    ```
    
3. **把它整段删掉！** 或者在前面加 `/*` 和 `*/` 把它注释掉。
    
4. _(注意：只删 `SysTick_Handler` 这一段，里面的其他东西比如 `NMI_Handler` 之类的千万别动。)_
## 难点代码解释

### 第一部分：打造上帝视角的“绝对时间”

```
// 定义全局时间戳
volatile uint32_t g_SystemTime = 0;
```

- **`uint32_t`**：这是一个 32 位的无符号整数。为什么要这么大？因为 1 秒钟它就会加 1000。如果用小的变量，很快就会“爆表”（溢出）。用 32 位，它可以连续记录 **49 天** 不溢出。
    
- **`g_`**：这是程序员的暗号，代表 Global（全局变量）。说明这个变量全公司（整个文件）哪都能看、哪都能用。
    
- **`volatile`**：**全场最关键的词！** 它是写给编译器看的。编译器有时候为了让程序跑得快，会把常用的变量死记在 CPU 的小本本（寄存器）里，不去内存里读。加了 `volatile` 就是警告编译器：“这个数字随时会被底层的硬件偷偷改变，你每次用到它，**必须老老实实去内存里现看**，绝不能偷懒凭记忆！”
    

```
// 在 main 函数开头调用这一行，配置为 1ms 进一次中断
SysTick_Config(SystemCoreClock / 1000); 
```

- **`SystemCoreClock`**：代表单片机的心跳频率，通常是 72,000,000（72MHz）。
    
- **`/ 1000`**：把 7200万 除以 1000，等于 72000。
    
- **整行意思**：告诉内核里的 SysTick 硬件定时器：“老兄，你去数 CPU 的心跳，**每数满 72000 下（正好是 1 毫秒的真实物理时间），你就给我拉响一次警报！**”
    

```
// SysTick 的专属中断服务函数（每 1ms 硬件自动调用一次）
void SysTick_Handler(void) {
    g_SystemTime++; // 墙上的挂钟滴答走了一秒
}
```

- **物理动作**：无论 CPU 在干什么牛逼的计算，只要 SysTick 的警报一响（每 1 毫秒一次），CPU 必须瞬间放下手头的工作，跑到这个函数里，把 `g_SystemTime` 加 1，然后瞬间跑回去继续刚才的工作。**这个过程极快，快到甚至不影响你的 `while(1)` 循环。**
    

---

### 第二部分：准备老板的“记事本”（状态机变量）

在死循环里，CPU 是没有记忆的。它要知道下一步该干嘛，全靠看这四个变量。

```
uint8_t  g_MotorState = 0;       // 电机当前状态：0停, 1加速, 2匀速, 3减速
```

- **作用**：机器的总指挥棒。它是 CPU 接下来要进哪扇门的钥匙。

```
uint16_t g_CurrentARR = 1200;    // 当前实时的速度 (ARR值)
uint16_t g_TargetARR  = 99;      // 想要达到的目标速度
```

- **作用**：因为我们不能像以前那样一次性把速度拉满，所以必须有两个变量。`g_CurrentARR` 记录“我现在走到哪一步了”，`g_TargetARR` 记录“我最终要走到哪”。

```
uint32_t g_LastStepTime = 0;     // 上一次微调速度的时间戳
```

- **作用**：**非阻塞延时的灵魂！** 它用来记录上一次 CPU 去“拧油门”的具体时间。
    

---

### 第三部分：主循环（狂奔的流水线）

```
while(1)
{
```

- **物理动作**：只要不断电，CPU 就在这个大括号里以每秒几百万圈的速度疯狂绕圈圈。
    

#### 任务 1：处理突发事件（接指令）

```
    if (g_ReceiveCmd != 0) 
    {
```

- **含义**：CPU 跑路过这里，看一眼有没有通过串口发来的新指令。如果没有（等于 0），**立刻跳过整个 `if` 块**，一微秒都不耽误。
    

```
        if(g_ReceiveCmd == 'A') { // 收到前进指令
            Motor_SetDir(1);
            g_CurrentARR = 1200;      // 从最慢速起步
            g_TargetARR = 99;         // 目标是 60转/分
            Motor_SetSpeed_ARR(g_CurrentARR);
            Motor_SetState(1);        // 启动脉冲
```

- **动作拆解**：收到 `'A'`，开始备车。设好方向，设定极慢的初始速度 1200（极其安全，绝不报警），设定目标速度 99，然后 `Motor_SetState(1)` 让底层硬件开始慢慢发脉冲，车子像蜗牛一样动起来了。
    
```
            g_MotorState = 1;         // 【切换状态】：进入“加速”状态
            g_LastStepTime = g_SystemTime; // 记录下起步的精确时间
        }
```

- **核心逻辑**：车子虽然动了，但我们还没开始加速。怎么让它加速？我们把总指挥棒 `g_MotorState` 掰到 `1`，然后**看一眼墙上的挂钟，把当前时间（比如 5000 毫秒）写进 `g_LastStepTime`**。然后 CPU 立刻离开这里！
    

```
        else if(g_ReceiveCmd == 'S') { // 收到紧急刹车指令
            g_TargetARR = 1200;       // 目标变成最慢速
            g_MotorState = 3;         // 【无缝切换状态】：直接进入“减速”状态！
        }
        g_ReceiveCmd = 0; // 清空指令
    }
```

- **强行打断**：如果收到 `'S'`，不管电机刚才在干嘛，直接把目标改成最慢速 1200，把指挥棒掰到 `3`。清空指令，转身就走。
    

---

#### 任务 2：执行状态机（看时间，干微操）

CPU 离开上面那段代码，来到了执行区。

```
    // 【状态 1：加速中】
    if (g_MotorState == 1) 
    {
```

- **门禁**：CPU 看一眼指挥棒，如果是 `1`，就进这扇门。如果刚才收到了 `'S'` 指挥棒变成了 `3`，这扇门就直接锁死，CPU 进不去。

```
        // 核心非阻塞判断：距离上次调速，过去 3 毫秒了吗？
        if (g_SystemTime - g_LastStepTime >= 3) 
        {
```

- **最难懂也最精妙的一行**：
    
    - `g_SystemTime` 是此刻墙上挂钟的时间（比如现在是 5001 毫秒）。
        
    - `g_LastStepTime` 是咱们起步时记下的时间（5000 毫秒）。
        
    - `5001 - 5000 = 1`。1 不大于等于 3。
        
    - **结果**：条件不成立！CPU 什么都不做，直接跳出 `if`，结束这一圈 `while(1)`，去跑下一圈！
        
    - **经过无数圈后**，挂钟终于走到了 5003 毫秒。`5003 - 5000 = 3`。条件成立！门开了！
        
```
            g_LastStepTime = g_SystemTime; // 刷新便签本上的时间
            g_CurrentARR -= 3;             // 速度加快一点点
            Motor_SetSpeed_ARR(g_CurrentARR);
```

- **动作拆解**：既然 3 毫秒到了，CPU 干三件事：
    
    1. 把挂钟上的 `5003` 重新抄写到本子上（覆盖掉原来的 5000）。**这意味着下次想进来，必须等挂钟走到 5006！**
        
    2. 把当前速度从 1200 减掉 3，变成 1197（ARR越小跑得越快）。
        
    3. 把 1197 丢给底层硬件执行。
        

```
            // 检查：达到目标速度了吗？
            if (g_CurrentARR <= g_TargetARR) {
                g_MotorState = 2; // 【切换状态】：加速完毕，进入“匀速”状态
            }
        }
    }
```

- **自我销毁**：每次加速完，检查一下。如果 `g_CurrentARR` 已经一路减到了 `99`，说明加速完成了！这时候把指挥棒 `g_MotorState` 掰到 `2`。
    
- **结果**：下一圈 CPU 跑到 `if (g_MotorState == 1)` 时，发现条件不满足了！这段加速代码彻底“隐身”，单片机保持 `99` 的速度平稳运行。
    

```
    // 【状态 3：减速中】
    else if (g_MotorState == 3)
    {
        if (g_SystemTime - g_LastStepTime >= 3) 
        {
            g_LastStepTime = g_SystemTime; 
            g_CurrentARR += 3;             // 速度减慢一点点 (ARR 变大)
            Motor_SetSpeed_ARR(g_CurrentARR);

            if (g_CurrentARR >= g_TargetARR) {
                Motor_SetState(0); // 速度降到了 1200，说明速度极慢了，彻底锁死
                g_MotorState = 0;  // 【切换状态】：把指挥棒掰回 0，进入“停机待命”
            }
        }
    }
}
```

- **减速逻辑**：完全同理。只是 `ARR` 变成每次加 3，直到加回 1200（最慢速），然后彻底切断脉冲发车 `Motor_SetState(0)`，最后把状态归零。
## 思考提问
如果我在代码执行到 `g_CurrentARR = 600`（加速到一半）的时候，突然触发了串口接收到 `'S'`，各个变量和代码走向会发生什么样的奇妙变化？

### 案发前 0.1 毫秒：电机正在狂奔

- **当前时间：** `g_SystemTime = 800`（系统刚跑了 800 毫秒）
    
- **指挥棒：** `g_MotorState = 1`（老板正在执行“加速”任务）
    
- **当前速度：** `g_CurrentARR = 600`（油门刚好踩到一半）
    
- **目标速度：** `g_TargetARR = 99`（原本打算一直加速到 99）
    

---

### 案发瞬间：突发事件降临

**第一步：秘书接电话（耗时 1 微秒）** 你在电脑上按下了字母 `'S'`。 底层的 USART1 硬件瞬间拉响警报，单片机立刻跳进中断函数。 秘书迅速把 `'S'` 放在了老板的桌面上： 👉 `g_ReceiveCmd = 'S'`

**第二步：老板低头看桌面（耗时 1 微秒）** 老板在跑 `while(1)` 循环，刚好路过“任务 1：处理指令”区。 他一看桌上是 `'S'`，二话不说，拿起笔强行篡改了便签本： 👉 **把目标速度划掉，改成最慢速：** `g_TargetARR = 1200` 👉 **把指挥棒“啪”地一下掰到 3：** `g_MotorState = 3`（直接宣布：现在进入减速状态！） 处理完，老板把桌上的 `'S'` 扔进垃圾桶（`g_ReceiveCmd = 0`），继续往下走。

---

### 见证奇迹的时刻：完美衔接

**第三步：经过加速区（完美避开）** 老板带着改好的便签本往下走，来到了刚才一直在跑的加速代码门前：

```
if (g_MotorState == 1) // 门禁检查
```

老板一看便签，现在的状态已经是 `3` 了！ **结果：** 砰！加速区的门瞬间对老板锁死。那句 `g_CurrentARR -= 3` 彻底失效。 **物理意义：** 电机在 `ARR=600` 的位置，**瞬间停止了加速**！绝对不会再往下踩一毫米油门。

**第四步：切入减速区（丝滑接管）** 老板继续往下走，来到了减速代码门前：
```
else if (g_MotorState == 3) // 门禁检查
```

验证通过！老板直接推门进去。 他看了一眼墙上的挂钟，只要距离上次踩油门过去 3 毫秒了，他就执行： 👉 `g_CurrentARR += 3`

**极其关键的一点来了：** 注意看，此时的 `g_CurrentARR` 是多少？ **它还是刚才的 `600`！** 老板并没有把它重置。 所以，老板是从 `600` 开始，把它变成了 `603`，然后是 `606`，`609`…… **物理意义：** 电机不会发出“咔哒”一声惨叫，也不会瞬间停转。它是从刚好加速到一半的状态，**顺势**开始变得越来越慢，直到 `ARR` 慢慢加回 `1200`，最后平稳停死！

### 总结：为什么 Delay 绝对做不到？

如果咱们用的是以前的 `for` 循环和 `Delay_ms`： 当你在 `ARR=600` 的时候按下 `'S'`。由于老板被锁死在 `Delay` 的睡眠状态里，他根本不知道你按了刹车。 他会像个瞎子一样，头铁地继续把 `ARR` 降到 99，让电机达到最高速，跑完这整个 1.1 秒的死循环之后，才猛然发现：“哎呀，刚才有个刹车指令！” 然后又花 1.1 秒减速。 在工业现场，这 1.1 秒的延迟，可能就意味着机器把人的手臂卷进去了。
