在实际的机械控制中，我们不可能让传送带永远以一个速度傻转。我们要学会在程序运行的过程中，随时随地做到三件事：**换挡（改速度）、打方向（正反转）、踩刹车（启停）**。
很多初学者就是因为没有深挖这三个指令的物理细节，直接把它们扔进代码里，结果导致电机**“经常莫名其妙卡死”**、**“换向的时候猛烈打齿”**或者**“调速的时候突然不转了”**。

这三大核心指令背后，其实各自隐藏着一个极度致命的“新手坑”。我们现在就把这三个坑全部刨出来填平！

---

### 第一坑：踩刹车 (`TIM_Cmd`) —— “锁死”还是“溜车”？

**表层代码：** `TIM_Cmd(TIM2, DISABLE);`

这行代码会让 TIM2 机器瞬间停机，PA0 不再发送脉冲。但是，电机停下之后的状态是什么？

**深挖细节：**

- **默认的“电子手刹”（Holding Torque）：** 在我们的接线中，驱动器的 `EN+` 和 `EN-` 是悬空什么都没接的。根据 UMLP57 手册，这意味着电机默认处于“使能”状态。当你停止发送脉冲时，电机虽然不转了，但驱动器依然会向电机线圈里通入巨大的电流。**此时电机的轴是被死死锁住的！** 你用手根本拧不动。对于传送带来说，这是好事，哪怕停在斜坡上东西也不会滑下来。
    
- **什么是“溜车”（脱机自由状态）？** 手册里明确写了，当 `EN+` 接 +5V，`EN-` 接低电平时，驱动器将切断电机各相的电流，使电机处于自由状态，此时步进脉冲不被响应 。
    
- **实战结论：** 用 `TIM_Cmd(TIM2, DISABLE)` 停车，是带着“刹车”停的。除非你想用手去拨动传送带，否则永远不要去接 `EN` 引脚。
    

---

### 第二坑：打方向 (`GPIO_SetBits`) —— 必须遵守的“交通规则”

**表层代码：** `GPIO_SetBits(GPIOA, GPIO_Pin_1);` (改变 PA1 的高低电平来实现换向)

**深挖细节：**

- **致命的 50 微秒规则：** 驱动器内部的芯片也是有反应时间的。UMLP57 手册中极其严格地规定：**方向控制信号应先于脉冲信号至少 50μs 建立** 。
    
- **如果你违规了会怎样？** 如果你一边发着脉冲让电机狂转，一边瞬间调用代码改变方向引脚，驱动器在收到方向改变的那一瞬间可能还在处理上一个正向脉冲，这会导致电机内部磁场瞬间错乱，产生剧烈的“咔哒”打齿声，甚至直接触发红灯报警！
    
- **实战结论（机械保护法则）：** **绝对不允许在电机高速运转时直接换向！**
    
    正确的换向逻辑必须是：
    
    1. 踩刹车停发脉冲：`TIM_Cmd(TIM2, DISABLE);`
        
    2. 等几毫秒让机械结构稳定。
        
    3. 打方向：`GPIO_SetBits` 或 `GPIO_ResetBits`。
        
    4. 重新启动：`TIM_Cmd(TIM2, ENABLE);`
        

---

### 第三坑：踩油门 (`TIM_SetAutoreload`) —— 最容易翻车的“半截脉冲”

这是坑死无数人的一个底层逻辑。

**表层代码：** 我们知道改速度就是改周期上限（ARR）。

比如原本 60转/分 的 ARR 是 99。想变成 120转/分，直接写：

`TIM_SetAutoreload(TIM2, 49);` （把周期缩短一半，速度翻倍）。

**深挖细节：**

你还记得我们之前是怎么捏出 50% 的完美方波的吗？

我们把 `TIM_Pulse`（对应底层的 `CCR1` 比较寄存器）写死了等于 **50**。

**灾难推演：**

1. 你的周期 ARR 变成了 **49**。
    
2. 但是你的脉冲宽度 CCR1 还是 **50**！
    
3. 定时器从 0 往上数，数到 49 就清零了，它**永远也数不到 50**！
    
4. 结果就是：PA0 引脚永远输出高电平，不再有高低起伏的方波。你的电机收到一个持续的死电平，会瞬间失去动力，“嗡”的一声卡死在原地。
    

- **实战结论（油门与离合必须配合）：** 每次改变速度（修改 ARR），**必须同时**修改占空比（CCR1），让它永远等于新 ARR 的一半！
    
    安全调速的代码必须是捆绑在一起的两句：
    
    ```
    TIM_SetAutoreload(TIM2, 49);                // 踩油门：修改总周期 (ARR)
    TIM_SetCompare1(TIM2, 49 / 2);              // 踩离合：同步修改脉冲宽度 (CCR1)
    ```
    

---

### 把这三个细节封装成“傻瓜式 API”

看懂了上面的三大深坑，我们就可以把它们封装成三个非常安全的自定义函数。以后在 `main` 函数里，你只需要像指挥员工一样调用它们就行了，再也不用管底层的死活。

把这三个函数加在你的代码里：

```
// 1. 安全启停控制 (state填1启动，填0停止)
void Motor_SetState(uint8_t state)
{
    if(state == 1) {
        TIM_Cmd(TIM2, ENABLE);  // 发脉冲
    } else {
        TIM_Cmd(TIM2, DISABLE); // 停脉冲，但电机保持锁死
    }
}

// 2. 安全方向控制 (dir填1正转，填0反转)
void Motor_SetDir(uint8_t dir)
{
    // 严谨的工控逻辑：换向必须先停机，保护机械结构！
    Motor_SetState(0); 
    
    // 给一点微小的延时（用空循环代替），满足手册里方向信号需提前 50us 建立的要求
    for(int i=0; i<1000; i++); 

    if(dir == 1) {
        GPIO_SetBits(GPIOA, GPIO_Pin_1);
    } else {
        GPIO_ResetBits(GPIOA, GPIO_Pin_1);
    }
}

// 3. 安全调速控制 (填入你想让单片机输出的 周期ARR 值)
// 比如填 99 就是 60转，填 49 就是 120转
void Motor_SetSpeed_ARR(uint16_t arr_value)
{
    TIM_SetAutoreload(TIM2, arr_value);       // 修改上限
    TIM_SetCompare1(TIM2, arr_value / 2);     // 永远保持 50% 的完美方波！
}
```

#具体执行的时候，遇到报警问题
在“动作二停稳后，刚进入动作三准备反转”时报警，凶手绝对是以下两个物理定律的其中之一（大概率是第二个）

### 凶手一：起步的“第一脚油门”还是太猛了（启动频率过高）

在之前的代码里，我们让它以 `ARR=400` 起步。

- 算一笔账：`ARR=400` 对应的脉冲频率是 6000Hz。经过 1600 细分和 1:15 减速箱换算后，**这相当于让电机在 0 秒内，瞬间从 0 蹦到 225转/分钟（电机本体转速）**！
把 `ARR`（周期上限）设为了 400（代码里是 `400 - 1`）。

意思是：机器每数 400 下，就发出去 1 个完整的脉冲。

**计算公式：**

$$频率 = \frac{2400000}{400} = 6000\text{ Hz}$$

**结论：** 单片机现在每秒钟向驱动器疯狂发射 **6000 个**脉冲。
驱动器侧面的拨码开关，被你设定成了 **1600 细分**。

这在物理上的意思是：**驱动器每收到 1600 个脉冲，电机本体才会刚好转完完整的一圈（360度）。**

那么，单片机一秒钟发了 6000 个脉冲，电机一秒钟能转几圈呢？

**计算公式：**

$$每秒转速 = \frac{6000}{1600} = 3.75\text{ 圈/秒}$$

工业上大家习惯用“转/分钟 (RPM)”来衡量速度，所以我们要乘上 60 秒：

$$每分钟转速 = 3.75 \times 60 = 225\text{ 转/分钟 (RPM)}$$

**结论：** 当 `ARR=400` 时，**电机本体**（没经过减速箱之前的那根主轴）的真实转速就是 **225 RPM**。
- **电机本体转速（225 RPM）：** 刚才算出来的 225转/分，是步进电机内部转子狂飙的速度。对于一个有重量的金属转子来说，在 **0 秒内瞬间起步**达到 225转，惯性太大了，电机受不了，所以驱动器红灯报警！
    
- **减速箱输出轴转速（传送带真实速度）：** 因为你加了一个 1:15 的涡轮减速箱，所以即使里面狂转 225 圈，外面的传送带每分钟实际只走了：
    
    $$输出轴转速 = \frac{225}{15} = 15\text{ 转/分钟}$$
    

### 总结物理法则公式

$$ARR = \frac{2400000 \times 60}{电机本体转速 \times 1600} = \frac{90000}{电机本体转速}$$
- 步进电机有一个物理特性叫“最大启动频率”。瞬间跳到 225转，转子依然会被“憋住”，导致闭环驱动器瞬间判定失步报警。
    
- **解决办法：** 把起步的 ARR 放得更大，比如 `ARR=1200`（瞬间起步速度降到极低）。

### 凶手二：120转/分 打破了步进电机的“物理天花板”（扭矩断崖）

这是最核心、最致命的原因！

你可能觉得传送带 120转/分 不算快，但别忘了你有一个 **1:15 的减速箱**！

- 传送带 120转/分钟 $\times$ 15倍减速比 = **电机本体需要狂飙到 1800转/分钟！**
    
- **工业常识：** 57系列步进电机（哪怕是闭环的）在 24V 供电下，最佳工作区间是 300~600转。一旦超过 1000转，它的扭矩就会呈**断崖式下降**（几乎变成 0）。
    
- **报警真相：** 你的电机在加速跑到 1000多转的时候，力量已经小到连自身的转子和减速箱齿轮都带不动了。它**物理上堵转了**，随后编码器立刻报警！
    

---

认清了物理规律，我们就得在代码里“敬畏物理”。

1. **起步极其温柔：** 所有起步从 `ARR=1200` 开始（极慢速起步）。
    
2. **放缓加速过程：** 把加速循环里的 `Delay_ms(2)` 改成 `Delay_ms(3)`，给转子更多跟上磁场的时间。
    
3. **降维打击（封印最高速）：** 把反转的最高速从 120转（1800电机转速，必死无疑）降到 **80转**（对应 ARR=74，电机转速 1200转，这已经是 24V 下 57电机的极限了）。



```
    while(1)
    {
        // ==========================================
        // 动作 1：【正转】60转/分 巡航
        // ==========================================
        Motor_SetDir(1);          
        
        // 1. 挂超低速一挡起步 (ARR=1200，绝不报警)
        Motor_SetSpeed_ARR(1200);  
        Motor_SetState(1);        

        // 2. 缓慢加速到 60转 (ARR=99)
        for(int arr = 1200; arr >= 99; arr -= 3) 
        {
            Motor_SetSpeed_ARR(arr); 
            Delay_ms(3);          // 加长延时，加速更柔和
        }

        // 3. 匀速巡航 3 秒
        Delay_ms(3000);           

        // 4. 缓慢刹车 (从 99 降速回到 1200)
        for(int arr = 99; arr <= 1200; arr += 3) 
        {
            Motor_SetSpeed_ARR(arr); 
            Delay_ms(3);
        }

        // 5. 彻底停稳
        Motor_SetState(0);        
        Delay_ms(1000);           


        // ==========================================
        // 动作 3：【反转】80转/分 极限速 (不敢再快了)
        // ==========================================
        Motor_SetDir(0);          
        
        // 1. 依然是超低速起步
        Motor_SetSpeed_ARR(1200);  
        Motor_SetState(1);        

        // 2. 加速到 80转 (ARR=74)
        // 注意：80转对应的电机转速是 1200转，这已经是极限，不能再压榨了
        for(int arr = 1200; arr >= 74; arr -= 3) 
        {
            Motor_SetSpeed_ARR(arr); 
            Delay_ms(3);
        }

        // 3. 极速巡航 1.5 秒
        Delay_ms(1500);           

        // 4. 缓慢刹车 (从 74 降速回到 1200)
        for(int arr = 74; arr <= 1200; arr += 3) 
        {
            Motor_SetSpeed_ARR(arr); 
            Delay_ms(3);
        }

        // 5. 彻底停稳
        Motor_SetState(0);        
        Delay_ms(1000);           
    }
```

### 第一步：算单片机发出的脉冲频率

单片机的倒计时速度是 2,400,000 Hz，现在每数 1200 下发一个脉冲。

$$频率 = \frac{2400000}{1200} = 2000\text{ Hz}$$

**结论：** 单片机一秒钟只发出 **2000 个**脉冲。（对比之前的 6000 个，温和了太多！）

### 第二步：算电机本体（转子）的速度

驱动器需要 1600 个脉冲才让电机转 1 圈。

$$每秒转速 = \frac{2000}{1600} = 1.25\text{ 圈/秒}$$

换算成分钟：

$$每分钟转速 = 1.25 \times 60 = 75\text{ 转/分钟 (RPM)}$$

**结论：** 当 `ARR=1200` 时，电机本体的转速只有区区的 **75 RPM**。这对于步进电机来说，完全处于扭矩最大、最舒服的“散步”区间。就算在 0 秒内瞬间启动到 75转，电机的绝对力量也能轻松克服静摩擦力，**绝对不会憋死报警**。

### 第三步：算减速箱输出轴（传送带）的真实速度

别忘了，电机本体转 15 圈，外面的传送带才转 1 圈。

$$输出轴转速 = \frac{75}{15} = 5\text{ 转/分钟}$$

```c
// ==========================================
// 1. 工具函数：粗略的毫秒级延时
// ==========================================
void Delay_ms(uint32_t ms)
{
    uint32_t i, j;
    for(i = 0; i < ms; i++)
        for(j = 0; j < 7200; j++); // 针对 72MHz 主频的粗略延时
}

// 2. 声明我们自己封装的“三大无敌 API”

// ==========================================
// 3. 总裁办：主函数 (编写流水线剧本)

int main(void)
{
    // 产线初始化，底层配置全部在这里面静悄悄地完成
    Motor_Init(); 

    // 死循环：流水线开始 24 小时无休止倒班
    while(1)

// 启停控制：1 启动，0 停止
void Motor_SetState(uint8_t state) {
    if(state == 1) TIM_Cmd(TIM2, ENABLE);
    else TIM_Cmd(TIM2, DISABLE);
}

// 方向控制：1 正转，0 反转
void Motor_SetDir(uint8_t dir) {
    Motor_SetState(0); // 绝对铁律：换向必须先停机！
    Delay_ms(2);       // 延时 2 毫秒。手册规定方向信号必须提前 50 微秒建立 ，2 毫秒绝对安全了。
    
    if(dir == 1) GPIO_SetBits(GPIOA, GPIO_Pin_1);
    else GPIO_ResetBits(GPIOA, GPIO_Pin_1);
}

// 调速控制：修改周期上限
void Motor_SetSpeed_ARR(uint16_t arr_value) {
    TIM_SetAutoreload(TIM2, arr_value);       // 修改大周期
    TIM_SetCompare1(TIM2, arr_value / 2);     // 同步修改占空比，保持 50% 方波
}

// 硬件初始化（之前的所有结构体代码全搬到这里了）
void Motor_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    TIM_TimeBaseInitTypeDef  TIM_TimeBaseStructure;
    TIM_OCInitTypeDef  TIM_OCInitStructure;

    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_AFIO, ENABLE);
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_0;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_OD;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_1;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_OD;
    GPIO_Init(GPIOA, &GPIO_InitStructure);
    
    GPIO_SetBits(GPIOA, GPIO_Pin_1); 

    TIM_TimeBaseStructure.TIM_Prescaler = 30 - 1;     
    TIM_TimeBaseStructure.TIM_Period = 100 - 1; // 默认给个 100 的初值
    TIM_TimeBaseStructure.TIM_ClockDivision = 0;
    TIM_TimeBaseStructure.TIM_CounterMode = TIM_CounterMode_Up;
    TIM_TimeBaseInit(TIM2, &TIM_TimeBaseStructure);

    TIM_OCInitStructure.TIM_OCMode = TIM_OCMode_PWM1;
    TIM_OCInitStructure.TIM_OutputState = TIM_OutputState_Enable;
    TIM_OCInitStructure.TIM_Pulse = 50; 
    TIM_OCInitStructure.TIM_OCPolarity = TIM_OCPolarity_High;
    TIM_OC1Init(TIM2, &TIM_OCInitStructure);

    TIM_OC1PreloadConfig(TIM2, TIM_OCPreload_Enable);
    TIM_ARRPreloadConfig(TIM2, ENABLE);
    // 注意：这里没有写 TIM_Cmd，因为我们要把启动权交给上面的流水线剧本
}
```
