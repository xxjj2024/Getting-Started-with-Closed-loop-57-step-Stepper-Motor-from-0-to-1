为了实现交替运行步数，引入定时器中断。
## 现实案例
链条节数交替。
### 真正的“物理-脉冲”换算

**第一步：算出输出轴转一圈的真实脉冲数**

电机本身是 1600 细分，加上 1:15 的减速箱，意味着电机内部必须狂转 15 圈，外面的驱动轴才会转 1 圈。

$$\text{输出轴单圈脉冲数} = 1600 \times 15 = 24000 \text{ 个脉冲}$$

**第二步：算出输出轴转一圈，链条走多远**

外接齿轮的直径是 127mm，我们求它的周长：

$$\text{齿轮周长} = \pi \times 127 \approx 398.98 \text{ mm}$$

_(物理意义：单片机每发出 24000 个脉冲，链条刚好往前推进 398.98 mm)_

**第三步：算出 1 毫米对应多少脉冲 (系统分辨率)**

$$\text{脉冲当量} = \frac{24000}{398.98} \approx 60.153 \text{ 脉冲/mm}$$

**第四步：算出 24 节和 26 节的终极目标脉冲**

根据咱们之前的图纸推导，150 节链条严丝合缝地铺在 1422.56 mm 的跑道上：

- 单节链条平均长度：$1422.56 \div 150 \approx 9.484 \text{ mm}$
    
- **走 24 节链条的物理距离：** $24 \times 9.484 \approx 227.61 \text{ mm}$
    
- **走 26 节链条的物理距离：** $26 \times 9.484 \approx 246.58 \text{ mm}$
    

将物理距离乘以脉冲当量，得出最终填入代码的硬核数据：

- **24 节对应脉冲：** $227.61 \times 60.153 \approx \mathbf{13691 \text{ 脉冲}}$
    
- **26 节对应脉冲：** $246.58 \times 60.153 \approx \mathbf{14832 \text{ 脉冲}}$

                        if(g_CurrentARR > g_TargetARR) g_MotorState = 1; 
                        else if(g_CurrentARR < g_TargetARR) g_MotorState = 3; 
                    }
                    break;
                    
                case 0x02: // 无限测速模式 (反转)
                    if(g_MotorState == 0) {
                        Motor_SetDir(0);
                        g_CurrentARR = 1200;  
                        g_TargetARR = g_ParsedARR; 
                        Motor_SetSpeed_ARR(g_CurrentARR);
                        Motor_SetState(1);    
                        g_MotorState = 1;

                    break;

                case 0x05:

                        g_CurrentARR = 600;  
                        Motor_SetSpeed_ARR(g_CurrentARR);

                    }
                    break;
            }

        }

// 【中断秘书办：底层脉冲计数与极速刹车】 (全篇最硬核新增)
// ==========================================
void TIM2_IRQHandler(void)
{
    // 检查是否是 TIM2 触发了更新中断（即刚发出了一个 PWM 脉冲）
    if (TIM_GetITStatus(TIM2, TIM_IT_Update) != RESET)
    {
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update); // 消除中断标志位
        
        // 只有在“精准定位模式”且状态为4时，才开启疯狂计数
        if (g_PositionMode == 1 && g_MotorState == 4) 
        {
            g_CurrentPulses++; // 脉冲计数器 +1
            
            // 如果走到了目标指定的脉冲数！
            if (g_CurrentPulses >= g_TargetPulses) 
            {
                TIM_Cmd(TIM2, DISABLE); // 【物理级强制拉闸停机】
                g_MotorState = 0;       // 机器回到待命状态
                g_PositionMode = 0;     // 定位任务圆满结束
            }
        }
    }
}

void Send_Status_Report(void) 
{
    g_ReportPacket[0] = 0xAA;
    g_ReportPacket[1] = 0x55;
    g_ReportPacket[2] = 0x81;                      
    g_ReportPacket[3] = g_MotorState;              
    g_ReportPacket[4] = (g_CurrentARR >> 8) & 0xFF;
    g_ReportPacket[5] = g_CurrentARR & 0xFF;       
    g_ReportPacket[6] = (uint8_t)(g_ReportPacket[2] + g_ReportPacket[3] + g_ReportPacket[4] + g_ReportPacket[5]);
    g_ReportPacket[7] = 0xFF;

    DMA_Cmd(DMA1_Channel4, DISABLE);           
    DMA_SetCurrDataCounter(DMA1_Channel4, 8);  
    DMA_Cmd(DMA1_Channel4, ENABLE);            
}

void USART1_IRQHandler(void)
{
    static uint8_t rx_state = 0;     
    static uint8_t rx_cmd = 0;       
    static uint8_t rx_arr_h = 0;     
    static uint8_t rx_arr_l = 0;     
    static uint8_t rx_checksum = 0;  

    if(USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)
    {
        uint8_t rx_data = USART_ReceiveData(USART1); 
        
        switch(rx_state) 
        {
            case 0: 
                if(rx_data == 0xAA) rx_state = 1; else rx_state = 0; break;
            case 1: 
                if(rx_data == 0x55) rx_state = 2; else if(rx_data == 0xAA) rx_state = 1; else rx_state = 0; break;
            case 2: 
                rx_cmd = rx_data; rx_state = 3; break;
            case 3: 
                rx_arr_h = rx_data; rx_state = 4; break;
            case 4: 
                rx_arr_l = rx_data; rx_state = 5; break;
            case 5: 
                rx_checksum = rx_data; rx_state = 6; break;
            case 6: 
                if(rx_data == 0xFF) 
                {
                    uint8_t calc_sum = (uint8_t)(rx_cmd + rx_arr_h + rx_arr_l);
                    if(calc_sum == rx_checksum) 
                    {
                        if(rx_cmd == 0x00) {
                            IWDG_ReloadCounter();
                        } else {
                            g_ParsedCmd = rx_cmd;
                            g_ParsedARR = (rx_arr_h << 8) | rx_arr_l; 
                            g_NewFrameReady = 1; 
                            IWDG_ReloadCounter();
                        }
                    }
                }
                rx_state = 0; break;
        }
        USART_ClearITPendingBit(USART1, USART_IT_RXNE);
    }
}

### 核心解密：TIM 更新中断 vs 普通中断

#### 1. 什么是普通中断（比如之前的串口中断）？

你可以把串口中断（`USART1_IRQHandler`）想象成**“收发室大爷”**。 大爷平时在打瞌睡，只有当外面真的来了快递（电脑发来数据），大爷才会被惊醒，爬起来去处理。**它是由外部不可控事件触发的。**

#### 2. 什么是 TIM2 的“更新中断 (Update Interrupt)”？

在咱们的代码里，TIM2 扮演的是**“机枪手”**。 它的工作原理是：内部有一个计数器，从 0 开始疯狂往上数，数到你设置的 `ARR` 值（比如 600），它就会打出一发子弹（输出一个 PWM 脉冲），然后**瞬间清零，重新从 0 开始数**。

这个“数到顶、清零重来”的瞬间，在 STM32 硬件底层就叫作一次 **“更新事件 (Update Event)”**。 如果你开启了更新中断，就等于给机枪手定了个铁规矩：**每打出去一发子弹，必须立刻大喊一声“报告”！**

#### 3. 为什么要用它？（致命的物理精度）

如果我们在主循环 `while(1)` 里去数脉冲，会发生什么灾难？ 主循环可能正在忙着算打包数据、算校验和。这时候 TIM2 在底层“突突突”打出了好几个脉冲，主循环根本来不及看，直接漏数了！这就导致你的托盘越跑越偏，最后直接撞毁。

而开启了 **更新中断**，意味着我们把“数脉冲”这个软件动作，**死死地、物理级地绑定在了硬件 PWM 波形上**。 只要硬件管脚吐出一个高低电平，CPU 无论在干什么，都会被瞬间强行打断，进入中断把计数器 `+1`。 **这叫作“硬实时 (Hard Real-time) 控制”，是绝对不会丢步、分毫不差的终极方案！**

### 代码逐行“显微镜”级拆解

现在我们带着上面的物理概念，来看看新加的代码每一行在干嘛：

#### 第一块：给狙击手配发的“瞄准镜” (全局变量)
```
volatile uint32_t g_TargetPulses = 0;    // 目标要走多少脉冲（比如 1828）
volatile uint32_t g_CurrentPulses = 0;   // 当前已经走了多少（从 0 开始算）
volatile uint8_t  g_PositionMode = 0;    // 模式开关：0=瞎跑，1=指哪打哪
```

- **大白话：** `volatile` 这个词极其关键！它是在警告编译器：“这三个变量随时会在中断里被偷偷改变，你每次用它们时，必须老老实实去内存里读，绝对不许做任何缓存优化！”如果不加，代码优化后可能会死机。
    

#### 第二块：赋予狙击手“最高特权” (`Motor_Init` 新增)
```
    // 开启 TIM2 的更新中断 (每打一发脉冲就喊报告)
    TIM_ITConfig(TIM2, TIM_IT_Update, ENABLE);

    // 分配秘书办，【极其致命的特权设置】
    NVIC_InitStructure.NVIC_IRQChannel = TIM2_IRQn;
    NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 0; // 抢占优先级 0 (全场最高！)
    NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;
    NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;
    NVIC_Init(&NVIC_InitStructure);
```

- **大白话：** 注意这里的 `PreemptionPriority = 0`（数字越小，官越大）。之前咱们串口中断的优先级是 1。 这意味着什么？**哪怕 CPU 正在处理电脑发来的串口数据，只要 TIM2 打出一个脉冲，CPU 必须立刻把手头的串口数据扔下，跑过去数脉冲！** 确保了脉冲计数绝对零延迟。
    

#### 第三块：指挥官下达开枪指令 (`main` 函数中的 `case 0x05`)
```

                    if(g_MotorState == 0) { // 只有在停机状态下才能接这个任务
                        Motor_SetDir(1);             // 挂上前进挡

                        g_CurrentPulses = 0;         // 已经走的步数清零
                        g_TargetPulses = g_ParsedARR;// 把电脑发来的 1828 填入目标
                        
                        // 直接给一个起步速度 (ARR=600)，不搞缓慢加速了
                        g_CurrentARR = 600;  
                        Motor_SetSpeed_ARR(g_CurrentARR);
                        
                        Motor_SetState(1);           // 物理拉闸，电机开始“突突突”！
                        g_MotorState = 4;            // 【神来之笔】
                    }
```

- **神来之笔的解释：** 为什么要凭空捏造一个 `g_MotorState = 4`？ 回想一下咱们的主循环，里面有一段“平滑调速”的代码，它一直在盯着状态 1（加速）和状态 3（减速）。 如果我们这里设为 1，主循环就会多管闲事，跑去改变我们设定好的 600 的速度。改成 4，主循环一看“不认识这个状态”，就不管它了，让电机以绝对稳定的速度匀速走到终点。
    

#### 第四块：狙击手无情扣动扳机 (`TIM2_IRQHandler`)
```
void TIM2_IRQHandler(void)
{
    // 检查：真的是 TIM2 打完了一发脉冲来报告的吗？
    if (TIM_GetITStatus(TIM2, TIM_IT_Update) != RESET)
    {
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update); // 消除报告声音，准备听下一次
        
        // 只有在咱们下达了定位任务时，才去数数
        if (g_PositionMode == 1 && g_MotorState == 4) 
        {
            g_CurrentPulses++; // 计数器 +1 (代表电机物理上往前走了一步)
            
            // 【全场最激动人心的判定】
            if (g_CurrentPulses >= g_TargetPulses) 
            {
                TIM_Cmd(TIM2, DISABLE); // 物理级切断电源，瞬间刹车！
                g_MotorState = 0;       // 汇报：已回到停机待命状态
                g_PositionMode = 0;     // 汇报：定位任务完美结束
            }
        }
    }
}
```

- **大白话：** 这就是整个精准走位的**“灵魂所在”**。CPU 就像一个极度专注的裁判，盯着电机发出的每一个脉冲。 走到 1827 步，继续！ 走到 1828 步，**条件成立，毫秒级瞬间执行 `TIM_Cmd(TIM2, DISABLE)`！** 57 闭环步进电机收到断电信号，利用其闭环特性死死锁在当前位置。那 24 节链条，不多一丝，不少一毫，完美停在操作位。
### 什么是 TIM2 更新中断？

为了彻底解决“老板走神导致漏数”的问题，STM32 芯片的设计师发明了**“硬件定时器中断（Update Interrupt）”**。

这相当于什么？ 相当于你给机器人的脚底下，直接绑了一个**“极其暴力的电击计步器”**。

1. **硬件自动生成（不需要老板管）：** TIM2 这个部门是一套纯硬件电路。只要你拉下电闸（`TIM_Cmd(ENABLE)`），它就在底层自动产生 PWM 脉冲（让机器人抬腿走路）。这个过程，CPU 老板是完全不需要参与的，老板甚至可以去睡觉。
    
2. **什么是“更新事件”？** 机器人的脚每一次落地（定时器计数器数满了，自动清零重新开始的那一瞬间），就是一个“更新事件”。
    
3. **什么是“更新中断”？** 你用代码 `TIM_ITConfig(TIM2, TIM_IT_Update, ENABLE);` 开启了这个功能。意思就是：机器人的脚**只要一落地**，计步器就会顺着电线，给老板的屁股上来一次高压电击！
    

### 完美的物理闭环（再看那段新代码）

现在，你带着“电击计步器”的思维，再来看这段最核心的代码：

```
// ==========================================
// 【这是老板被电击后的本能反应！】
// ==========================================
void TIM2_IRQHandler(void)
{
    // 1. 老板被电醒，确认一下：“卧槽，是不是机器人的脚又落地了？”
    if (TIM_GetITStatus(TIM2, TIM_IT_Update) != RESET)
    {
        // 2. 擦把汗，把电击器复位，准备挨下一次电。
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update); 
        
        // 3. 看看现在是不是“指哪打哪”的任务？
        if (g_PositionMode == 1 && g_MotorState == 4) 
        {
            // 4. 机器人确定走了一步，本子上立刻狠狠记上一笔！
            g_CurrentPulses++; 
            
            // 5. 【极其致命的判断】看看本子上的数字，到没到 1828 步？
            if (g_CurrentPulses >= g_TargetPulses) 
            {
                // 到了 1828 步的瞬间！不管老板手头在干嘛，立刻一刀切断机器人的电源！
                TIM_Cmd(TIM2, DISABLE); 
                
                // 任务完美结束，打卡下班。
                g_MotorState = 0;       
                g_PositionMode = 0;     
            }
        }
    }
}
```

### 为什么它能做到“分毫不差”？

因为你设置了 `PreemptionPriority = 0`（全场最高抢占优先级）。 这意味着，哪怕老板正在和顺丰快递员聊天（正在处理串口数据），只要机器人的脚一落地（触发 TIM2 中断），老板会瞬间一把推开快递员，在 1 微秒之内掏出本子写上 `+1`，然后再回头继续和快递员聊天。
